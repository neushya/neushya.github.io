# Zenito Markdown Web Version - Development History (Part 1)
**Date:** 2026-05-28 ~ 2026-05-29
**Order:** 01

## 1. 🚀 Project Vision & Pipeline
- **Goal:** Transform the Tauri-based desktop app into a professional-grade web service.
- **Pipeline Established:** 
  1. Requirements Analysis
  2. Architecture & UX Design
  3. Development Planning (Tech Stack)
  4. Iterative Development (Agile)

## 2. 📝 Stage 1: Requirements Analysis
### Key Decisions:
- **Storage Strategy:** **Option A (Local-First)**. Use `File System Access API` to directly read/write local files, providing a desktop-like experience.
- **Workspace Management:** Allow users to approve an entire **Workspace Folder** for hierarchical file access in the LNB.
- **Performance:** Maintain high-end **Pretty Print** for developers while simplifying the search UI for the web environment.

## 3. 🏛 Stage 2: Architecture Design
### Core Technical Stack:
- **Frontend:** React 19 + Vite + Tailwind CSS v4.
- **File I/O:** Replace Rust backend commands with browser-native `File System Access API`.
- **PWA (Progressive Web App):** Enable "Install" feature and offline support for a desktop-app feel.
- **Persistence:** Use `IndexedDB` (via `idb-keyval`) to store folder handles, allowing "Re-authorize Workspace" on reload.

## 4. 🛠 Stage 3: Initial Implementation Plan
### Design Preferences:
- **Landing Page:** "Open Folder" button on a clean Darcula screen (Identical to Desktop).
- **Sync Policy:** **Manual Refresh** for local file changes to optimize performance.
- **Icons:** Use the same premium icon set as the desktop version.
- **Isolation:** Create a completely independent project directory (`/NEW-DEV/md-editor-web`).
