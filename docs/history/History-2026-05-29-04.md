# Zenito 마크다운 웹 버전 - 개발 히스토리 (Part 4)
**날짜:** 2026-05-29
**저장 순서:** 04

## 1. 📂 Stage 4: 실전 운영을 위한 정밀 튜닝 착수
- **목표:** 웹 브라우저 환경에서의 데이터 안정성 확보 및 사용자 경험(UX) 극대화.
- **분석 테마:** 자동 저장 메커니즘, 폴더 트리 성능, PWA 설치 완성도.

## 2. 📝 자동 저장(Auto-save) 및 초안(Draft) 시스템 설계
- **결정 사항:** 팝업 없는 매끄러운 작업을 위해 '하이브리드 저장 방식' 채택.
- **신규 파일(Untitled):** 실제 파일 위치가 지정되기 전까지는 **IndexedDB**에 실시간으로 내용을 저장하여 브라우저 종료 시에도 데이터 복구 가능.
- **기존 파일:** 타이핑 중단 후 **2초(Debounce)** 뒤에 로컬 파일에 자동으로 쓰기 실행.
- **성능 최적화:** 비동기 처리를 통해 대용량 파일 저장 시에도 에디터 끊김 현상 방지.

## 3. 🌲 폴더 트리 및 성능 분석 결과
- **분석 내용:** 폴더 깊이(Depth) 제한에 따른 성능 실효성 검토.
- **결과:** 현재 적용된 **지연 로딩(Lazy Loading)** 방식은 클릭 시에만 하위 항목을 읽어오므로 깊이에 따른 성능 저하가 거의 없음이 확인됨.
- **최종 결정:** 사용자 편의를 위해 인텔리제이와 동일한 **무제한 폴더 계층 구조**를 유지함.

## 4. ✨ PWA 및 UI 고도화
- **설치 경험:** 주소창 내 설치 버튼 활성화를 위한 매니페스트 및 전용 아이콘 배치.
- **검색 UI:** 기존 GNB 위치를 유지하며 웹에 최적화된 심플한 검색 엔진 적용.
