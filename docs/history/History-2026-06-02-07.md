# Zenito 마크다운 웹 버전 - 개발 히스토리 (Part 7)
**날짜:** 2026-06-02
**저장 순서:** 07

## 1. 🛡️ 제니토하네스(Zenito Harness) 가동
- **정의:** 요구사항분석 > 상세설계 > 상세기획 > 개발계획 > 개발 > 검증으로 이어지는 전문가용 소프트웨어 공정의 명칭을 **'제니토하네스'**로 명명함.
- **적용:** 향후 모든 신규 기능 및 타 프로젝트 진행 시 이 파이프라인을 표준으로 준수함.

## 2. 📝 Stage 1: 모바일 호환성 요구사항 분석 결과
### 모바일 전용 심플 모드 확정:
- **파일 시스템:** 모바일 브라우저의 제약을 고려하여 **'단일 파일 업로드 및 다운로드'** 방식으로 단순화.
- **UI 레이아웃:** 좁은 화면을 효율적으로 쓰기 위해 에디터와 프리뷰를 50:50이 아닌 **'토글 버튼 기반 1:1 교체'** 방식으로 결정.
- **접근성:** 모바일 키보드 간섭 최소화 및 터치 친화적 인터페이스 구축.

## 3. 🏛️ Stage 2: 상세 설계 방향 (Architecture)
- **환경 감지:** 뷰포트 너비 및 UserAgent를 통한 Mobile/Desktop 자동 감지 로직 설계.
- **컴포넌트 분기:** 
    - `MobileView`: 사이드바 제거, 하단/상단 토글 바 추가.
    - `DesktopView`: 기존 3단 구성 유지.
- **데이터 흐름:** `File System Access API` 미지원 시 일반 `<input type="file">`을 활용한 Fallback 로직 설계.
