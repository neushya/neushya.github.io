## 1. 🚀 프로젝트 비전 및 파이프라인 수립
- **목표:** Tauri 기반 데스크톱 앱을 전문가급 웹 서비스로 전환.
- **개발 파이프라인:** 
  1. 요구사항 분석 (완료)
  2. 아키텍처 및 UX 기획 (완료)
  3. 상세 개발 계획 (진행 중)
  4. 반복적 개발 및 검증 (예정)

## 2. 📝 Stage 1: 요구사항 분석 결과
### 주요 결정 사항:
- **저장 전략:** **옵션 A (로컬 전용)**. 브라우저의 `File System Access API`를 사용하여 사용자의 PC 파일을 직접 읽고 쓰며, 데스크톱 앱과 동일한 경험 제공.
- **작업 공간 관리:** 사용자가 승인한 특정 **작업 폴더(Workspace)**를 기준으로 LNB에 전체 파일 트리를 노출.
- **성능 및 기능:** 개발자용 **Pretty Print** 성능은 그대로 유지하되, 검색 기능은 웹 환경에 맞춰 심플하게 구현.

## 3. 🏛 Stage 2: 기술 아키텍처 설계
### 핵심 기술 스택:
- **프론트엔드:** React 19 + Vite + Tailwind CSS v4.
- **파일 I/O:** Rust 백엔드 대신 브라우저 네이티브 `File System Access API` 도입.
- **PWA (Progressive Web App):** 설치 기능 및 오프라인 모드 지원으로 데스크톱 앱과 유사한 독립 실행 환경 제공.
- **영속성 관리:** `IndexedDB`를 사용하여 페이지 새로고침 후에도 작업 폴더 권한을 쉽게 재승인받는 시스템 설계.

## 4. 🛠 Stage 3: 초기 구현 계획 및 설정
### 디자인 및 UX 상세:
- **첫 화면:** 데스크톱과 동일하게 "Open Folder" 버튼이 중앙에 배치된 깔끔한 Darcula 화면.
- **동기화:** 사용자가 직접 새로고침을 누를 때만 로컬 변경 사항을 반영 (성능 최적화).
- **자산 활용:** 기존 데스크톱 프로젝트의 아이콘 및 UI 컴포넌트 소스를 복사하여 이식하기로 결정.
- **독립성:** 신규 폴더 `/NEW-DEV/md-editor-web`에서 완전히 새롭게 시작.
