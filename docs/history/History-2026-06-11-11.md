## 2. 결함 수정: 모바일 다크모드 Editor 상하 스크롤 불가

- **환경:** 모바일 Chrome + 다크모드 + 대용량 파일
- **증상:** Editor 화면에서 좌우 스크롤은 되지만 상하 스크롤 불가. 라이트모드는 정상 동작.
- **원인 분석:**
  - 라이트 테마(`lightTheme`)는 `"&": { height: "100%" }` 명시 → `.cm-editor` 높이 제한 → 세로 스크롤 작동.
  - 다크 테마(`oneDark`)는 `height` 미설정 → `.cm-editor`가 컨텐츠 높이만큼 팽창 → `.cm-scroller`가 `height: 100%`이지만 부모가 팽창해 overflow 미발생 → 스크롤 불가.
  - 부모 레이아웃의 `overflow-hidden`이 팽창한 내용을 clip하여 스크롤 없이 잘린 화면만 표시.
  - 좌우 스크롤이 동작하는 이유: `.cm-scroller`의 `overflowX: auto`는 height 제약과 무관하게 작동.
- **수정 내용 (`src/components/Editor.tsx`):**
  - 컬러 테마와 분리된 `baseLayoutTheme` 추가: `EditorView.theme({ "&": { height: "100%" } })`
  - `extensions` 배열에 `baseLayoutTheme`을 항상 포함 (다크/라이트 무관하게 적용)
  - `lightTheme`의 `&` 셀렉터에서 중복된 `height: "100%"` 제거
- **효과:** PC 다크모드의 잠재적 동일 결함도 함께 해결.

## 3. 기능 개선: PC 탭바 overflow 드롭다운

- **요청 사항:** 탭이 탭 영역을 초과할 때 드롭다운 버튼(`>>`)으로 숨겨진 탭 접근 가능하도록 개선.
- **주요 설계 결정:**
  - 탭 수량: 컨테이너 너비 기반 **동적 계산** (고정 개수 아님)
  - 활성 탭: 항상 탭바에 표시 (드롭다운으로 밀리지 않음)
  - `>>` 버튼 위치: 마지막 visible 탭 바로 다음 (absolute 우측 고정)
  - 드롭다운에서 탭 닫기: 패널 유지, overflow 탭 0개 시 자동 닫힘
- **구현 방식 (`src/components/TabBar.tsx`):**
  - 모든 탭을 항상 flex 흐름 안에 렌더링 (DOM `offsetLeft` 측정 정확도 보장)
  - overflow 탭: `visibility: hidden + pointer-events: none` (흐름 유지, 비가시화, 비상호작용)
  - overflow 감지: `tab.offsetLeft + tab.offsetWidth > container.clientWidth - DROPDOWN_BTN_W`
  - `useLayoutEffect`: 탭/activeTabId 변경 시 브라우저 페인트 전 재계산 (깜빡임 방지)
  - `ResizeObserver`: 창 크기 조절, 사이드바 토글 시 자동 재계산
  - 드롭다운 패널: `position: fixed + getBoundingClientRect()` 사용 → 부모 `overflow: hidden` 우회
- **수정 중 발생한 버그 이력:**
  - 1차: overflow 탭에 `absolute` 클래스 적용 → flex 흐름 이탈 → `offsetLeft = 0` → overflow 감지 즉시 해제되는 루프 발생 → 아무것도 변하지 않는 것처럼 보임
  - 2차: 드롭다운 패널을 `absolute top-full`로 배치 → `tabsAreaRef`의 `overflow: hidden`에 의해 잘림 → 버튼 클릭해도 드롭다운 미노출
  - 3차(현재): `position: fixed` + `getBoundingClientRect()`로 해결
