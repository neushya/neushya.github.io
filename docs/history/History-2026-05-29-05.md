# Zenito 마크다운 웹 버전 - 개발 히스토리 (Part 5)
**날짜:** 2026-05-29
**저장 순서:** 05

## 1. 📂 PDF 전용 뷰어 도입 결정
- **배경:** 텍스트 에디터에서 PDF를 열 경우 깨진 문자가 노출되고, 자동 저장 시 파일이 파손되는 리스크가 확인됨.
- **결정:** 단순 텍스트 로딩이 아닌, 실제 문서 내용을 확인할 수 있는 **전용 PDF 뷰어(Embedded Viewer)** 방식을 채택함.

## 2. 📝 PDF 처리 기술 설계
- **파일 오픈:** `showOpenFilePicker`의 허용 목록에 `application/pdf` 추가.
- **렌더링 방식:** PDF 선택 시 브라우저의 `Blob URL`을 생성하여 `iframe`을 통해 네이티브 PDF 뷰어를 화면에 렌더링함.
- **데이터 보호:** 
    - PDF 탭은 **'읽기 전용(Read-Only)'**으로 고정.
    - 자동 저장 엔진에서 PDF 탭을 제외하여 원본 파일 훼손을 원천 차단함.

## 3. 🛠 세부 구현 계획
- **탭 상태 확장:** `Tab` 인터페이스에 `isPdf` 플래그 추가.
- **UI 전환:** 활성화된 탭이 PDF일 경우 에디터/프리뷰 패널 대신 전체 화면 뷰어 노출.
- **리소스 관리:** 탭 폐기 시 생성된 `Blob URL`을 해제하여 메모리 누수 방지.
