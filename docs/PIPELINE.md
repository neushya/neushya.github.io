# 🛡️ 제니토하네스 (Zenito Harness Standard)

이 문서는 모든 프로젝트 및 신규 기능 개발 시 준수해야 할 표준 엔지니어링 파이프라인인 **'제니토하네스'**의 공정을 정의합니다.

---

## 🏗️ Stage 1: 요구사항 분석 (Analysis)
사용자의 비즈니스 니즈를 기술적 제약 사항과 대조하여 최적의 범위를 확정합니다.
1.  **기능 명세화:** 핵심 동작(MVP) 및 예외 상황 정의.
2.  **기술 타당성:** 플랫폼별(웹/모바일/데스크톱) API 지원 여부 분석.
3.  **리스크 검토:** 데이터 안전성 및 성능 영향도 보고.

## 🏛️ Stage 2: 상세 설계 (Detailed Design)
시스템 아키텍처와 데이터 흐름을 설계합니다.
1.  **컴포넌트 구조:** 모듈성 및 재사용성을 고려한 설계.
2.  **데이터 모델링:** 상태 관리(State) 및 영속성(Persistence) 구조 정의.
3.  **환경 분기:** 플랫폼별 특화 로직(Desktop vs Mobile) 분리.

## 📝 Stage 3: 상세 기획 (Planning)
사용자가 직접 경험할 UI/UX 시나리오를 구체화합니다.
1.  **사용자 시나리오:** 페이지 진입부터 완료까지의 모든 흐름 정의.
2.  **인터랙션 기획:** 터치, 키보드, 단축키 등 반응 효과 정의.
3.  **UI 프로토타이핑:** 화면 배치 및 시각적 일관성 검토.

## 📅 Stage 4: 개발 계획 (Dev Roadmap)
효율적인 구현을 위한 작업 순서와 우선순위를 정합니다.
1.  **작업 쪼개기:** 작은 단위의 개발 티켓 생성.
2.  **타임라인:** 개발 및 자가 검증 일정 수립.

## 💻 Stage 5: 정밀 개발 (Development)
실제 코드를 작성하고 유닛 테스트를 병행합니다.
1.  **Surgical Implementation:** 정밀한 코드 주입 및 리팩토링.
2.  **타입 무결성:** TypeScript 기준 엄격 준수.

## 🚀 Stage 6: 통합 검증 (Verification)
최종 동작을 확인하고 실서비스에 배포합니다.
1.  **QA:** 환경별(OS/Browser) 교차 테스트.
2.  **Deployment:** GitHub Actions 기반 자동 배포 및 라이브 모니터링.
3.  **History-Keeping:** 모든 과정을 히스토리 파일로 자산화.
