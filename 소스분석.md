## 3. App.tsx — 상태 관리 중추

### 3-1. State 전체 목록

| 상태 | 타입 | 기본값 | 설명 |
|------|------|--------|------|
| `isMobile` | boolean | `window.innerWidth < 768` | 모바일 여부 |
| `tabs` | Tab[] | `[]` | 열린 파일 탭 목록 |
| `activeTabId` | string\|null | `null` | 현재 활성 탭 ID |
| `rootHandle` | FileSystemDirectoryHandle\|null | `null` | 선택된 작업 폴더 핸들 |
| `rootName` | string | `"No Folder"` | 폴더 이름 표시용 |
| `viewMode` | 'split'\|'editor'\|'preview' | `'split'` | PC 뷰 모드 |
| `sidebarWidth` | number | `200` | 사이드바 너비(px) |
| `isPrettyPrint` | boolean | `true` | JSON/코드 포맷팅 활성화 |
| `lastActivePanel` | 'editor'\|'preview' | `'editor'` | 마지막 활성 패널 |
| `isSaving` | boolean | `false` | 저장 중 상태 표시 |
| `isInitialized` | boolean | `false` | IndexedDB 초기화 완료 여부 |
| `shortcuts` | ShortcutItem[] | DEFAULT_SHORTCUTS | 단축키 설정 |
| `isShortcutModalOpen` | boolean | `false` | 단축키 모달 표시 |
| `isThemeModalOpen` | boolean | `false` | 테마 모달 표시 |
| `isDarkMode` | boolean | `true` | 다크모드 여부 |

### 3-2. Tab 인터페이스

```typescript
interface Tab {
  id: string;                          // Math.random().toString(36).substring(7)
  name: string;                        // 파일명 (예: "index.md")
  path: string | null;                 // 상대 경로 (표시용)
  handle: FileSystemFileHandle | null; // 파일 시스템 핸들
  content: string;                     // 파일 내용 (PDF는 ObjectURL)
  isDirty: boolean;                    // 수정 여부 (* 표시)
  isPdf?: boolean;                     // PDF 파일 여부
}
```

### 3-3. 상수

```typescript
const APP_VERSION = "v0.1.4";
const STORAGE_KEY = "md_editor_shortcuts";       // localStorage 키
const WORKSPACE_KEY = "md_editor_workspace_handle"; // IndexedDB 키
const TABS_STATE_KEY = "md_editor_tabs_state";   // IndexedDB 키
const AUTO_SAVE_DELAY = 1000;                    // 자동저장 딜레이(ms)
```

### 3-4. 주요 로직 흐름

**초기화 (`useEffect` — mount 1회)**
```
1. matchMedia로 모바일 감지 → setIsMobile
2. IndexedDB에서 WORKSPACE_KEY → rootHandle 복원
3. IndexedDB에서 TABS_STATE_KEY → tabs + activeTabId 복원
   - PDF 탭은 content 비움 (ObjectURL 만료 대비)
4. isInitialized = true
```

**파일 열기 (`handleFileOpen`)**
```
1. 확장자 추출
2. PDF → URL.createObjectURL() → isPdf: true 탭 추가
3. 미디어/바이너리 확장자 → 차단 (alert)
4. 중복 파일 → 기존 탭 활성화
5. 텍스트 → FileSystemService.readFile() → 탭 추가
```

**자동 저장 (`updateContent`)**
```
1. 내용 변경 → isDirty: true
2. handle 있고 !isMobile → 1초 디바운싱
3. executeSave() → FileSystemService.writeFile()
4. 성공 시 isDirty: false, 0.5초 후 isSaving: false
```

**저장 분기 (`handleSave`)**
```
- isMobile: true  → handleExport() (Blob 다운로드)
- handle 있음    → executeSave()
- handle 없음    → showSaveFilePicker() 다이얼로그
```

**데이터 영속성**
```
localStorage  : isDarkMode, shortcuts (STORAGE_KEY)
IndexedDB     : rootHandle (WORKSPACE_KEY), tabs 상태 (TABS_STATE_KEY)
탭 변경 시     : isInitialized 체크 후 IndexedDB 자동 동기화
```

## 4. PC / Mobile 레이아웃 분리 구조

### 4-1. 3중 방어 분리 전략

단순 CSS 미디어 쿼리가 아닌 **JS 감지 + 조건부 렌더링 + CSS 강제 숨김** 3겹 구조.

#### Layer 1 — JavaScript 런타임 분기 (`App.tsx:285~299`)

```typescript
if (isMobile) {
  return (
    <div key={`mobile-root-${APP_VERSION}`} className="mobile-only-root h-[100dvh]">
      <MobileLayout ... />
    </div>
  );
}

return (
  <div key={`desktop-root-${APP_VERSION}`} className="pc-layout-root ...">
    {/* PC 전용 UI */}
  </div>
);
```

- `isMobile = true`이면 PC 코드가 DOM에 아예 존재하지 않음
- `key`에 `APP_VERSION` 포함 → 버전 업데이트 시 강제 재마운트 (캐시 파괴)

#### Layer 2 — CSS 강제 숨김 (`index.css:79~83`)

```css
@media (max-width: 767px) {
  .pc-layout-root {
    display: none !important;
  }
}
```

JS 분기 실패 시 2차 차단. `!important`로 오버라이드 방지.

#### Layer 3 — 동적 리사이즈 감지 (`App.tsx:78~86`)

```typescript
window.matchMedia("(max-width: 767px)").matches
window.addEventListener('resize', checkMobile);
```

창 크기 변경 시 실시간으로 레이아웃 전환.

### 4-4. PC vs Mobile 기능 비교

| 기능 | PC | Mobile |
|------|----|----|
| 폴더 열기 | FileSystem API (`showDirectoryPicker`) | **차단** + alert 안내 |
| 파일 저장 | `showSaveFilePicker` 다이얼로그 | `handleExport` (Blob 다운로드) |
| 자동 저장 | 1초 디바운싱 활성화 | **비활성화** (`!isMobile` 조건) |
| 새 파일명 | `"Untitled.md"` | `"Mobile-Draft.md"` |
| PDF 뷰 | `<iframe>` 인라인 렌더링 | `window.open('_blank')` |
| 뷰 모드 | Editor / Split / Preview 3종 | Editor / Preview 탭 토글 |
| 사이드바 | 파일 트리 표시 | 없음 |
| 상단 메뉴 | GNB (드롭다운 메뉴) | 아이콘 버튼 바 |
| 단축키 | GNB에 표시 | 없음 |
| Safe Area | 없음 | `pt/pb-[env(safe-area-inset-*)]` |
| 검색 대상 | `lastActivePanel` 기준 | `activeView` 기준 |
| 탭 닫기 확인 | `confirm()` 다이얼로그 | 생략 (`!isMobile` 조건) |

| | PC | Mobile |

## 6. 데이터 흐름

```
사용자 액션
     ↓
GNB / Sidebar / TabBar / MobileLayout (이벤트 발생)
     ↓ 콜백 호출
App.tsx (React State 업데이트)
     ↓ props 전달
Editor / Preview (리렌더링)
     ↓ 동시에
IndexedDB / localStorage (영속성 저장)
```

**파일 열기 전체 흐름**:
```
GNB.onOpenFile 클릭
→ window.showOpenFilePicker()
→ handleFileOpen(파일 핸들)
→ FileSystemService.readFile()
→ setTabs(새 탭 추가)
→ setActiveTabId(새 탭 ID)
→ Editor + Preview 자동 업데이트
```

**내용 변경 → 자동 저장 흐름 (PC만)**:
```
Editor onChange
→ updateContent(newContent)
→ setTabs(isDirty: true)
→ clearTimeout + setTimeout(1000ms)
→ executeSave()
→ FileSystemService.writeFile()
→ setTabs(isDirty: false)
→ setTimeout(500ms) → isSaving: false
```
